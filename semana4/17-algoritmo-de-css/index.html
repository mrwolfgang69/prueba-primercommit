Algoritmo de CSS

El Algoritmo de CSS hace referencia a la forma en que el navegador aplica los estilos al documento HTML. Es vital entender este concepto para que comprendas cómo se aplican y en algunas ocasiones cómo se sobreescriben las reglas CSS.

Tipos de algoritmos CSS

Cascada

La Cascada es el mecanismo en que el navegador aplica los estilos. Y toma en cuenta lo siguiente en este orden:

  1. El User Agent (los estilos por defecto del navegador)
  2. Los estilos especificados por el usuario.
  3. La especificidad de las reglas.

La cascada en las hojas de estilo significa que el orden de las reglas importa en CSS: cuando dos reglas tienen la misma especificidad, se aplica la que aparece en último lugar en el CSS.

Especificidad

La especificidad mide cuán específica es la selección de un selector. La Especificidad se calcula de la siguiente forma:

Etiquetas y pseudoelementos 0, 0, 0, 1
Clases, atributos y pseudoclases 0, 0, 1, 0
Idetificadores 0, 1, 0, 0
Estilos en línea 1, 0, 0, 0

La declaración !important rompe la especificidad. La única manera de anular la declaración !important sería incluir otra declaración !important que aparezca más adelante.

Herencia en CSS

La herencia también debe entenderse en este contexto: algunos valores de las propiedades CSS que se han establecido para los elementos padre los heredan los elementos hijo, pero otros no.
